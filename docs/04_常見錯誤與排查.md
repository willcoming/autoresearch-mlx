# 04｜常見錯誤與排查

---

## 1. `Could not find market data CSV`
### 原因
`data/GOOGL_1d_5y.csv` 不存在。

### 解法
把 CSV 放到：
```text
data/GOOGL_1d_5y.csv
```

---

## 2. `CSV missing required columns`
### 原因
CSV 欄位名稱不對。

### 正確 header
```csv
date,open,high,low,close,volume
```

---

## 3. `Need more data`
### 原因
資料長度不夠，切不出足夠的 train / validation / locked test。

### 解法
準備接近 5 年的日 K。

---

## 4. agent 開始改 `prepare.py`
### 問題
這會讓不同實驗不可比較。

### 解法
直接提醒它：
```text
Do not edit prepare.py. Revert any prepare.py changes and continue with train.py only.
```

---

## 5. agent 一次改太多
### 問題
你不知道到底是哪個改動造成結果變化。

### 解法
改用：
- `prompts/02_單次安全實驗.txt`

它會強制一次只做一個 bounded experiment。

---

## 6. Sharpe 上升，但 turnover 爆增
### 問題
可能只是靠高換手硬擠出表面績效。

### 解法
同時看：
- `val_sharpe`
- `median_turnover`
- `median_drawdown`

必要時回退。

---

## 7. 開了兩個 thread 同時改 `train.py`
### 問題
很容易互相覆蓋。

### 解法
- 同一個 checkout 一次只讓一個 thread 動
- 想平行就用 worktree

---

## 8. thread 太長，Codex 開始忘記規則
### 解法
- 開新 thread
- 或先讓它總結目前最佳版本，再繼續

可用提示詞：
```text
Summarize the current best train.py configuration, the latest kept metrics, and the top 3 next experiments. Do not edit files.
```

---

## 9. 想跑 final test，但不確定現在能不能跑
先用：
- `prompts/06_最終鎖定測試前檢查.txt`

等它確認：
- 規則沒被破壞
- `prepare.py` 沒亂動
- 搜尋已經收斂

再跑：
```bash
REVEAL_FINAL_TEST=1 uv run train.py > final_test.log 2>&1
grep "^locked_test_" final_test.log
```

---

## 10. Codex app 用 Cloud 跑不動本機訓練
### 原因
這個模板預設是跟本機 `autoresearch-mlx` / MLX 工作流搭配。

### 解法
訓練請改用：
- Codex app 的 Local
- 或直接用 Codex CLI
