# 01｜快速開始

這份文件給你最短可執行路徑。

## A. 前置條件
建議環境：

- Apple Silicon Mac
- Python 3.10+
- `uv`
- Git
- Codex app 或 Codex CLI
- 一份本地的 `GOOGL` 5 年日 K CSV

---

## B. 取得上游 repo
```bash
git clone https://github.com/trevin-creator/autoresearch-mlx.git
cd autoresearch-mlx
git checkout -b autoresearch/googl-sharpe-mar27
```

---

## C. 備份原始檔
```bash
cp prepare.py prepare.py.bak
cp train.py train.py.bak
cp program.md program.md.bak
```

---

## D. 複製本壓縮包內容
把以下檔案複製到 repo root：

- `prepare.py`
- `train.py`
- `program.md`
- `AGENTS.md`
- `docs/`
- `prompts/`
- `scripts/`
- `references/`
- `results.tsv`

若 `data/` 不存在就建立：
```bash
mkdir -p data logs
```

---

## E. 放資料檔
把真實資料放到：
```text
data/GOOGL_1d_5y.csv
```

欄位必須是：
```csv
date,open,high,low,close,volume
```

---

## F. 安裝依賴並跑 baseline
```bash
uv sync
uv run train.py > run.log 2>&1
./scripts/show_metrics.sh run.log
python scripts/append_result.py run.log keep "baseline"
```

若要看完整 log：
```bash
tail -n 80 run.log
```

---

## G. 基線通過後做什麼
### 用 Codex app
先貼：
- `prompts/01_檢查環境與基線.txt`

再貼：
- `prompts/02_單次安全實驗.txt`

### 用 CLI
```bash
./scripts/codex_exec_once.sh prompts/02_單次安全實驗.txt
```

---

## H. 最後才跑 locked final test
```bash
REVEAL_FINAL_TEST=1 uv run train.py > final_test.log 2>&1
grep "^locked_test_" final_test.log
```

---

## I. 最容易犯的錯
### 1) CSV 欄位不對
檢查 header 是不是：
```csv
date,open,high,low,close,volume
```

### 2) 資料不夠長
至少要接近 5 年日 K，否則 validation / locked test 不夠切。

### 3) 讓 agent 改了 `prepare.py`
這會破壞可比較性。搜尋期間只准改 `train.py`。

### 4) 同時開兩個 thread 改同一個 checkout
很容易互相覆蓋。要平行就用 worktree。
